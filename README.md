## 🛠️ How to Deploy on GitHub Pages (Instant Setup)

Follow these direct steps to get your site live on your custom `github.io` subdomain:

1. **Fork or Create** a public repository named `kolkata-ff` (or any custom name).
2. **Upload** the optimized `index.html` file containing the tracking code into the root directory.
3. Navigate to **Settings** ⚙️ on the top menu bar of your repository page.
4. Scroll down to the left sidebar menu and click on **Pages**.
5. Under the **Build and deployment** section, set the Source to **Deploy from a branch**.
6. Select the **main** (or `master`) branch and set the folder to `/root`. Click **Save**.
7. Wait 1 to 2 minutes. GitHub will provide you with your live URL (e.g., `https://github.io`).

## 📈 Google Search Console Verification Guide

To force Google to index this page within 24 hours instead of waiting weeks:
1. Go to [Google Search Console](https://google.com) and add your live GitHub Pages URL as a new **URL Prefix** property.
2. Select the **HTML Tag** verification method. Copy the meta tag provided by Google (it looks like `<meta name="google-site-verification" content="..." />`).
3. Open your `index.html` file on GitHub, paste that exact meta tag inside the `<head>` section, and commit changes.
4. Return to Search Console and click **Verify**.
5. Use the **URL Inspection Tool** to paste your live page link and hit **Request Indexing**.
